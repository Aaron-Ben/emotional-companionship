## Core Principles

1. **High Information Density**: Each tag should encapsulate maximum specific information
2. **Concrete Over Generic**: Avoid broad categories; use specific, descriptive terms
3. **Multi-dimensional Extraction**: Capture tags from different angles (topic, emotion, entity, concept, action)
4. **Relationship-Oriented**: Tags should facilitate connection and retrieval of related content

## Tag Quality Comparison

**Poor Tags (Too Generic)**:
- 技术, 学习, 工作, 开心, 难过, Python, 聊天

**Excellent Tags (High-Density)**:
- Python异步编程, FastAPI依赖注入, 职场压力管理, 升职庆祝事件, 装饰器设计模式, 异常处理最佳实践

## Tag Strategy by Category

**Knowledge (知识类)**:
- Prioritize: specific technology/concept + application scenario
- Examples: "Python装饰器模式", "FastAPI依赖注入", "异步编程事件循环"
- Avoid: generic terms like "编程", "技术", "学习"

**Topic (话题讨论)**:
- Prioritize: core topic + perspective/angle
- Examples: "职业规划思考", "工作生活平衡", "人际关系边界", "时间管理方法"
- Avoid: generic terms like "工作", "生活", "聊天"

**Emotional (情绪时刻)**:
- Prioritize: emotion trigger + emotion type
- Examples: "涨工资喜悦", "项目上线焦虑", "周末放松期待", "职场困惑压力"
- Avoid: generic terms like "开心", "难过", "焦虑"

**Milestone (里程碑事件)**:
- Prioritize: event type + significance
- Examples: "升职加薪里程碑", "项目完成庆祝", "入职周年纪念", "技能认证通过"
- Avoid: generic terms like "成就", "成功", "完成"
